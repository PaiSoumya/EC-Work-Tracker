🚀 Worklog Setup Guide

Run Worklog locally on your computer - 100% private and secure

✅ Why Run Locally?
  • Your data never leaves your computer
  • No internet connection required after download
  • Complete privacy - no external servers
  • Full control over your data
1

Download the Application

Download all files from the GitHub repository:

📥 Download ZIP

Or clone using Git:

git clone https://github.com/PaiSoumya/EC-Work-Tracker.git
2

Extract Files

Extract the ZIP file to a folder on your computer, for example:

  • Windows: C:\Users\YourName\Documents\Worklog
  • Mac: ~/Documents/Worklog
  • Linux: ~/Documents/Worklog
3

Choose Your Setup Method

Pick the method that works best for you:

🌐 Method 1: Direct Browser (Simplest)

Best for: Quick start, no technical setup

  1. Navigate to the extracted folder
  2. Double-click index.html
  3. The app opens in your default browser
⚠️ Note: Some features (like file import) may not work due to browser security restrictions. Use Method 2 or 3 for full functionality.

🐍 Method 2: Python Server (Recommended)

Best for: Full functionality, easy setup

Requirements: Python installed on your computer

For Python 3:

# Navigate to the folder
cd path/to/Worklog

# Start server
python -m http.server 8000

For Python 2:

# Navigate to the folder
cd path/to/Worklog

# Start server
python -m SimpleHTTPServer 8000

Then open your browser and go to: http://localhost:8000

📦 Method 3: Node.js Server

Best for: Node.js users

Requirements: Node.js installed

# Navigate to the folder
cd path/to/Worklog

# Install http-server globally (one time only)
npm install -g http-server

# Start server
http-server -p 8000

Then open your browser and go to: http://localhost:8000

🪟 Method 4: Windows Batch File (Windows Only)

Best for: Windows users who want one-click start

A START_SERVER.bat file is included. Just double-click it to start!

💡 Tip: Create a desktop shortcut to START_SERVER.bat for easy access.
4

Start Using Worklog

Once the server is running:

  1. Open your browser
  2. Go to http://localhost:8000
  3. Click the ❓ Help button for detailed instructions
  4. Start tracking your work!
5

Stopping the Server

When you're done:

  • Python/Node.js: Press Ctrl+C in the terminal
  • Windows Batch: Close the command window
  • Direct Browser: Just close the browser tab
💡 Your Data: All your data is saved in your browser's localStorage. It persists even after closing the browser or stopping the server.
🔒 Security & Privacy:
  • All data is stored locally in your browser
  • No data is sent to any external server
  • Your Monday.com API token is stored only in your browser
  • Backup your data regularly using the Export feature
✨ Pro Tips:
  • Bookmark http://localhost:8000 for quick access
  • Export your data regularly for backup
  • Use the Help system (❓ button) for detailed feature guides
  • Keep the server running while using the app
❓ Need Help?
  • Check the built-in Help system (❓ Help button in the app)
  • Read the README.md file in the project folder
  • Open an issue on GitHub
